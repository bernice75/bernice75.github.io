# Back-end Engineer | 김미림

구조적 결함과 처리 모델의 한계를 분석하고, 이를 재설계로 해결해 온 **4년차 백엔드 개발자 김미림**입니다.

Reactive 체인 붕괴로 장애가 반복되던 TCP 서버를 재설계해 리소스 고갈을 근본 차단했고, 주문/재고/외부 연동이 결합된 복합 트랜잭션 구조를 이벤트 기반 모델로 전환해 시스템 결합도를 낮추고 처리 안정성을 확보했습니다. 기능 추가보다 트랜잭션 경계, 리소스 관리 전략을 먼저 정의하는 개발을 지향합니다.

최근에는 LLM 기반 서비스 구조와 RAG 아키텍처를 스터디하며, 기존 시스템 위에 AI 기능을 안정적으로 결합하기 위한 설계 방식에 관심을 두고 있습니다.
